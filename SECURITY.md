# Security Policy

We actively maintain and provide security updates for the following versions of InputBooster:

| Version | Minecraft | Supported          |
| ------- | --------- | ------------------ |
| 2.0.2   | 1.21.x    | :white_check_mark: (Latest) |
| 2.0.1   | 1.21.x    | :white_check_mark: |
| 2.0.0   | 1.21.x    | :white_check_mark: |
| 1.x.x   | 1.20.x    | :x:                |
| < 1.0   | Various   | :x:                |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability in InputBooster, please help us protect our users by following these steps:

### Where to Report

**DO NOT** open a public GitHub issue for security vulnerabilities.

Instead, report privately via:
- **GitHub Security Advisories:** [Report a vulnerability](https://github.com/ahaduzzamankhan/inputbooster/security/advisories/new)
- **Email:** bdgamer9191@gmail.com with subject line `[SECURITY] InputBooster Vulnerability`

### What to Include

Please provide as much detail as possible:
- Description of the vulnerability
- Steps to reproduce the issue
- Affected versions
- Potential impact (client crash, data leak, privilege escalation, etc.)
- Any suggested fixes or patches

### What to Expect

- **Initial Response:** Within **48 hours** of your report
- **Status Updates:** Every **5-7 days** until resolved
- **Resolution Timeline:** Critical issues within **7 days**, others within **30 days**

### Response Process

1. **Acknowledgment** — We'll confirm receipt of your report
2. **Investigation** — We'll verify and assess the severity
3. **Fix Development** — We'll develop and test a patch
4. **Disclosure** — We'll release the fix and credit you (unless you prefer anonymity)

### If Accepted
- We'll release a security patch as soon as possible
- You'll be credited in the release notes (optional)
- A security advisory will be published on GitHub

### If Declined
- We'll explain why the issue doesn't qualify as a vulnerability
- We may still address it as a regular bug fix
- You're welcome to discuss our decision privately

## Security Best Practices

When using InputBooster:
- ✅ **Download only from official sources** (Modrinth, GitHub Releases, CurseForge)
- ✅ **Verify file hashes** before installing
- ✅ **Keep Fabric API and Minecraft updated** to the latest stable versions
- ✅ **Report suspicious behavior** immediately
- ❌ **Never download from unofficial mod repost sites**

---

## Scope

### In Scope
- Client-side crashes or exploits
- Memory leaks or resource exhaustion
- Configuration file injection attacks
- Malicious input handling

### Out of Scope
- Issues in Fabric API or Minecraft itself (report to Mojang/Fabric)
- Theoretical vulnerabilities with no proof of concept
- Social engineering attacks
- Third-party mod conflicts
