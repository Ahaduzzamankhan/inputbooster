# Contributing to InputBooster

Thanks for your interest in contributing to **InputBooster**! We love contributions from the community—whether it's bug fixes, new features, or ideas to improve the performance and responsiveness of low-FPS Minecraft gameplay.

Please follow these guidelines to help us maintain a friendly and organized project.  

---

## 1. Code of Conduct
- Be **respectful and friendly** to all contributors.  
- Harassment, hate speech, or toxic behavior **will not be tolerated**.  
- For details, see our [Contributor Covenant](https://www.contributor-covenant.org/).  

---

## 2. How to Contribute
### Reporting Bugs
- Open an **issue** in GitHub with a clear title and description.  
- Include steps to reproduce the bug.  
- Mention your **Minecraft version, mod version, and any other mods** you are using.  

### Suggesting Features
- Open an **issue** and describe your idea clearly.  
- Explain **why this feature is useful** and how it might work.  

### Submitting Code
- **Fork** the repository.  
- Create a new **branch** for your changes (`feature/your-feature` or `bugfix/your-bug`).  
- Write **clean, readable code** following Java naming conventions.  
- Test your changes in Minecraft before submitting.  
- Submit a **pull request** with a clear description of your changes.  

---

## 3. Pull Request Guidelines
- Each PR should address **one feature or bug** at a time.  
- Include screenshots or short videos if your PR affects visuals or gameplay.  
- Follow the **existing code style** and structure.  
- Be open to feedback and make changes if requested by maintainers.  

---

## 4. Other Ways to Contribute
- **Documentation:** Help improve our README, wiki, or guides.  
- **Testing:** Try new builds and report bugs.  
- **Community Support:** Help answer questions in forums or Discord.  

---
